# Παραγωγή Κανόνων από Βάσεις Δεδομένων Σκακιστικών Φινάλε με Μεθόδους Μηχανικής Μάθησης

Ο κώδικας που συνοδεύει τη διπλωματική εργασία του **Κωνσταντίνου Φραγκούδη**, Τμήμα Ηλεκτρολόγων Μηχανικών και Τεχνολογίας Υπολογιστών, Πανεπιστήμιο Πατρών, 2026.

**Επιβλέπων:** Καθηγητής Κυριάκος Σγάρμπας

---

## Περιεχόμενο της εργασίας

Ο κώδικας υλοποιεί μια ενιαία μεθοδολογία εξαγωγής ερμηνεύσιμων σκακιστικών κανόνων από endgame tablebases, χρησιμοποιώντας δέντρα απόφασης. Η μεθοδολογία εφαρμόζεται σε τρία θεμελιώδη φινάλε:

- **KPK** — Βασιλιάς και Πιόνι εναντίον Βασιλιά
- **KRK** — Βασιλιάς και Πύργος εναντίον Βασιλιά
- **KQK** — Βασιλιάς και Βασίλισσα εναντίον Βασιλιά

Για κάθε φινάλε υπάρχουν έξι αρχεία Python που υλοποιούν το ενιαίο pipeline:

| Αρχείο | Ρόλος |
|--------|-------|
| `Generator.py` | Παραγωγή του dataset μέσω probing στις Syzygy tablebases |
| `DecTree.py` | Εκπαίδευση και αξιολόγηση δέντρου απόφασης |
| `Comp.py` | Σύγκριση raw / geometric / strategic features |
| `RuleConsolidation.py` | Συμπύκνωση κανόνων σε σκακιστικά motifs |
| `Visual.py` | Παραγωγή γραφημάτων και οπτικοποιήσεων |
| `ReadMe.py` | Παραγωγή αυτόματης τεκμηρίωσης εκτέλεσης |

Συνολικά, το repository περιλαμβάνει 18 αρχεία Python (6 ανά φινάλε), οργανωμένα ενιαία και ακολουθώντας την ίδια αρχιτεκτονική.

## Ρύθμιση των Syzygy Tablebases

Ο κώδικας απαιτεί **τοπική εγκατάσταση** των Syzygy tablebases για φινάλε 3-4-5 κομματιών (περίπου 1 GB συνολικά).

### Βήμα 1: Κατέβασμα

Κατεβάστε τα αρχεία από τον παρακάτω σύνδεσμο:

**https://chess.massimilianogoi.com/download/tablebases/**

Επιλέξτε το **3-4-5 pieces Syzygy tablebases** archive. Θα χρειαστείτε μόνο αυτό — τα αρχεία 6 και 7 κομματιών δεν είναι απαραίτητα για KPK, KRK και KQK.

### Βήμα 2: Αποσυμπίεση

Αποσυμπιέστε τα αρχεία σε έναν φάκελο της επιλογής σας. Καλή πρακτική είναι να είναι σε **SSD** (όχι κλασικό μηχανικό δίσκο) ώστε το probing να είναι γρήγορο.

### Βήμα 3: Ενημέρωση διαδρομών στα scripts

Σε κάθε ένα από τα τρία Generator scripts (`KPKGenerator.py`, `KRKGenerator.py`, `KQKGenerator.py`), θα βρείτε στην αρχή του αρχείου μια μεταβλητή σαν αυτή:

```python
SYZYGY_PATH = Path(r"C:\path\to\syzygy\3-4-5")
```

**Αλλάξτε την** ώστε να δείχνει στον φάκελο όπου έχετε αποθηκεύσει τα Syzygy αρχεία. Παραδείγματα:

- **Windows:** `Path(r"C:\Users\YourName\Downloads\Syzygy\3-4-5")`
- **Linux:** `Path("/home/yourname/syzygy/3-4-5")`
- **macOS:** `Path("/Users/yourname/syzygy/3-4-5")`

### Βήμα 4: Ενημέρωση φακέλων εξόδου

Σε **όλα τα scripts** (και τα 18), θα βρείτε μια μεταβλητή σαν αυτή:

```python
KPK_OUTPUT_ROOT = Path(r"C:\path\to\output\KPK Dataset")
```

Αλλάξτε την ώστε να δείχνει σε έναν φάκελο όπου έχετε δικαίωμα εγγραφής. **Το ίδιο path πρέπει να χρησιμοποιείται και στα 6 scripts του ίδιου endgame** (π.χ. όλα τα KPK scripts πρέπει να δείχνουν στον ίδιο φάκελο).

## Σειρά εκτέλεσης

Για κάθε φινάλε ξεχωριστά, τα scripts πρέπει να τρέξουν με την παρακάτω σειρά:

1. **`Generator.py`** — Παράγει το dataset. Χρειάζεται πρόσβαση στις Syzygy. Τρέχει μία φορά.
2. **`DecTree.py`** — Εκπαιδεύει το κύριο μοντέλο και εξάγει μετρικές.
3. **`Comp.py`** — Τρέχει τη σύγκριση των τριών επιπέδων features.
4. **`RuleConsolidation.py`** — Συμπυκνώνει τους κανόνες σε motifs.
5. **`Visual.py`** — Παράγει γραφήματα και οπτικοποιήσεις.
6. **`ReadMe.py`** — Παράγει αυτόματη τεκμηρίωση εκτέλεσης.

Αφού τρέξει ο `Generator.py` **μία φορά**, τα υπόλοιπα scripts μπορούν να τρέξουν πολλές φορές χωρίς επανάληψη του probing — διαβάζουν απευθείας το παραγόμενο dataset.

Παράδειγμα εκτέλεσης για το KPK:

```
python KPKGenerator.py
python KPKDecTree.py
python KPKComp.py
python KPKRuleConsolidation.py
python KPKVisual.py
python KPKReadMe.py
```

Η ίδια λογική ακολουθείται και για KRK, KQK.

## Δομή εξόδου

Κάθε endgame παράγει έναν φάκελο εξόδου με την ακόλουθη δομή:

```
KPK Dataset/
├── 01_dataset/                    (τα CSV του Generator)
├── 02_model_wdl/                  (κύριο μοντέλο, μετρικές, confusion matrix)
├── 03_feature_set_comparison/     (σύγκριση raw/geometric/strategic)
├── 04_figures/                    (γραφήματα PNG)
└── 05_rule_consolidation/         (motifs, signatures, coverage)
```

Η δομή είναι πανομοιότυπη και για τα τρία endgames (με το `02_model_wdl` να αντικαθίσταται από `02_model_dtz` στα KRK και KQK).

## Απαιτήσεις υπολογιστικού χρόνου

Ενδεικτικοί χρόνοι εκτέλεσης σε σύγχρονο υπολογιστή (SSD, i7, 16 GB RAM):

- **Generator:** μερικά λεπτά ανά endgame (εκτελεί εκατοντάδες χιλιάδες probes)
- **DecTree + Comp:** συνολικά περίπου 5-10 λεπτά ανά endgame
- **RuleConsolidation, Visual, ReadMe:** λιγότερο από 1 λεπτό το καθένα

Ο πλήρης κύκλος και για τα τρία endgames ολοκληρώνεται τυπικά σε μια ώρα.
